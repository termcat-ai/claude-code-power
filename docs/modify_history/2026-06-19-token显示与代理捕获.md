# 2026-06-19 修改摘要：Token 显示 + 本地代理捕获完整 LLM 请求

## 一句话概述

为 claude_code_power 插件增加了每次提问的 Token 消耗展示（上行/下行/缓存命中），并实现了本地 HTTP 反向代理功能，可完整捕获发给 Anthropic API 的原始请求（含 system prompt 和 tool definitions）与原始 SSE 响应，并在历史面板中提供分 tab 的原始数据查看器。

## 背景 / 目标

- **Token 展示**：用户希望直观看到每次提问消耗了多少 token，包括上行（input）、下行（output）、缓存命中（cache read/write）。
- **原始请求查看**：Claude Code 的 JSONL 文件不存储 system prompt 和 tool definitions，无法从日志重建完整请求；用户希望能看到实际发给大模型的完整 payload。
- **代理捕获方案**：通过在 localhost 启动 HTTP 反向代理并注入 `ANTHROPIC_BASE_URL`，实现对 Claude Code 所有 API 调用的透明捕获，兼容有/无自定义 `baseUrl` 的 preset。

## 关键决策与理由

### Token 计算方式：取"最后一条"而非求和

- **问题**：一次用户提问可能产生多次 API 调用（tool-use 轮次）。每次调用的 `input_tokens` 包含全量 context，若累加会严重重复计数。
- **决策**：`inputTokens`、`cacheReadTokens`、`cacheWriteTokens` 用"最后一条 assistant event"的值覆盖（而非 max 或 sum）；只有 `outputTokens` 累加（每次调用产出都是新增量）。
- **被否决**：用 max 取值 —— 仍然不正确，当 context window 增长时不同调用的绝对值不同，max 并不代表"实际消耗"。

### 图标选择：`code` 而非 `braces`

- `braces` 不在 `termcat_client/src/plugins/ui-contribution/utils/icon-resolver.tsx` 的 ICON_MAP 白名单里，被静默跳过（按钮渲染为空），导致按钮不可见。
- 改为 `code`（即 `</>` 图标），已在白名单中。

### 原始弹窗结构：一次调用一个 tab

- **被否决（方案 A）**：两个固定 tab（"所有上行" / "所有下行"），多次调用的内容各自拼接 —— 用户反馈看不清哪次调用对应哪次响应。
- **被否决（方案 B）**：上行和下行用分隔线拼在一个文本框里显示 —— 用户反馈需要分开显示。
- **最终方案**：单次调用 → 2 个 tab（上行 / 下行）；多次调用（含 tool-use 轮次）→ N 个 tab（"调用 1"、"调用 2"...），每 tab 内上行 + 下行拼合，分隔线分开。

### SSE 响应存储：保留原始文本，不重建 message 对象

- 早期版本 `parseSseToMessage()` 将 SSE 流重建为 message JSON，但用户需要"保持原始文本"。
- 改为直接存储 raw SSE string（`rawResponseSse`）；仅保留 `extractMessageId()` 用于与 JSONL 的 `msg_xxx` 关联。

### 代理端口：OS 随机分配

- 使用 `server.listen(0, '127.0.0.1')` 让 OS 分配空闲端口，完全避免端口冲突。
- 被否决：固定端口 18764 —— 可能已被占用。

### 截断去掉，改为复制按钮

- 早期 modal 截断 500KB，用户反馈"内容被截断"。
- 改为不截断（直接传全文），弹窗 tab 栏右侧加"复制"按钮，由用户自行复制全文。
